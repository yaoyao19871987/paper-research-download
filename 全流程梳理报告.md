# 论文检索与下载全流程梳理报告

## 1. 当前项目实际上分成两套系统

### A. CNKI 研究流水线

目录：

`D:\Code\paper-download\Chinese paper search\cnkiLRspider`

它负责：

1. 接收主题
2. 用 Kimi + DeepSeek 开两轮专家会
3. 生成知网专业检索式
4. 在 CNKI 内执行多轮搜索
5. 抓取结果卡片、去重、打分、摘要分类
6. 生成候选下载清单

主入口：

- `research_pipeline.py`

核心依赖文件：

- `kimi_client.py`：双专家讨论、提示词、模型调用
- `cnki_common.py`：Selenium 驱动 CNKI 搜索页
- `journal_tiers.py`：期刊等级映射
- `scoring.py`：候选论文打分

### B. 思学图书馆 / 下载控制链

目录：

`D:\Code\paper-download\src`

它负责：

1. 登录思学图书馆
2. 人工输入验证码
3. 保存会话
4. 在浏览器里半自动点击、跳转、下载
5. 在下载页动态等待 PDF/CAJ 准备好

主入口：

- `src/live-session.js`
- `src/save-session.js`
- `src/download-paper.js`

注意：这套 JS 链路目前**还没有和研究流水线自动打通**。也就是说，研究流水线会给出“该下哪些论文”，但不会自动把这些论文送进思学图书馆代理下载。

## 2. 研究流水线的真实阶段

### 阶段 1：选题会 / 专家会

脚本：

- `research_pipeline.py`
- `kimi_client.py`

输出：

- `strategy_round1.json`
- `expert_discussion_round1.json`
- `expert_discussion_round1.md`

作用：

先把主题拆成一条核心检索式和多条补充检索式，避免进知网之后“乱搜”。

### 阶段 2：知网 Round 1 搜索

脚本：

- `research_pipeline.py`
- `cnki_common.py`

动作：

1. 打开 CNKI 高级/专业检索页
2. 切到学术期刊库
3. 关闭扩展检索等偏召回设置
4. 输入 round1 检索式
5. 依次按默认、被引、下载三个排序视角抓取
6. 每个视角最多抓 `CNKI_MAX_PAGES_PER_VIEW` 页

输出：

- `papers_master_raw.csv`
- `papers_master.csv`

### 阶段 3：摘要批量分析

脚本：

- `research_pipeline.py`
- `kimi_client.py`
- `scoring.py`

动作：

1. 将新抓到的论文按 batch 送给 Kimi
2. 判断它是历史研究、教学文、混合文还是未知
3. 给出保留理由和相关度
4. 结合期刊等级、被引、下载量、年份打总分

输出：

- `kimi_map_batches.jsonl`
- `papers_selected.csv`

### 阶段 4：Round 2 交叉验证 / 二轮策略

脚本：

- `research_pipeline.py`
- `kimi_client.py`

动作：

1. 根据第一轮抓到的文献热点和噪音分布
2. 重新开一次专家会
3. 生成第二轮补充检索式
4. 再跑一轮知网搜索

输出：

- `strategy_round2.json`
- `expert_discussion_round2.json`
- `expert_discussion_round2.md`

### 阶段 5：候选下载清单

脚本：

- `research_pipeline.py`

动作：

1. 从高分且值得保留的论文中选前 N 条
2. 生成人工勾选清单

输出：

- `papers_for_download.csv`
- `candidate_review.md`

### 阶段 6：最终下载队列

脚本：

- `research_pipeline.py`

动作：

1. 你手工在 `papers_for_download.csv` 中把 `user_select` 标成 `yes`
2. 再次运行流水线
3. 生成最终下载队列

输出：

- `download_queue.csv`

### 阶段 7：思学图书馆 / VPN 代理下载

当前状态：

这一步**还没有自动接上**。

现在已有能力：

1. `src/save-session.js`：登录思学图书馆并保存会话
2. `src/live-session.js`：进入人工指挥模式
3. `src/download-paper.js`：在单个下载页等待并保存文件

现在缺少能力：

1. 从 `download_queue.csv` 自动驱动思学图书馆入口
2. 自动把论文标题带入代理 CNKI 再搜一遍
3. 自动从结果页跳到 papermao 下载页
4. 自动批量消化下载队列
