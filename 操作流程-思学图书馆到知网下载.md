# 操作流程（固化版）

思学图书馆 -> 文献下载 -> 入口1 -> 检索1 -> 代理 CNKI -> 下载页 -> 文件落盘

这份文档描述的是当前项目建议保留的“老路径”。

## 1. 登录

1. 打开 `http://www.sixuexiazai.com/`
2. 优先复用已保存登录态
3. 如果登录态失效：
   - 自动填写账号密码
   - 截取文本验证码
   - 调用 Kimi OCR 识别
   - 失败则刷新验证码重试
4. 进入会员首页

## 2. 入口链路

1. 点击 `文献下载`
2. 点击 `入口1`
3. 在 papermao 认证页点击 `检索1`
4. 新开代理 CNKI 页面

如 `入口1` 异常，可人工切到其他入口，但自动化默认优先 `入口1`。

## 3. 代理 CNKI 检索规范

1. 进入代理 CNKI 后，固定切换检索字段到 `篇关摘`
2. 输入检索词
3. 点击检索
4. 在结果页优先命中目标题名
5. 从结果页进入 `cdown` 下载页

## 4. 下载页规则

当前真实下载页通常不是老式 `PDF1/PDF2` 页面，而是 `papermao cdown` 页面。

常见按钮优先级：

1. `下载2`（通常是 PDF）
2. `点此下载`
3. `移动(HW)`
4. `下载1`

说明：

- 页面会显示倒计时，例如 `本页面 97 秒后自动关闭`
- 页面脚本可能会自动 `window.close()`
- 自动化不能把“页面自动关闭”误判成失败
- 应该以浏览器上下文里的下载事件为准

## 5. 人类化节奏

为了提高稳定性，关键点击前后都要加入随机停顿。

建议节奏：

- 点击前：`1-3` 秒
- 点击后：`1-3` 秒
- 页面跳转后观察：`2-5` 秒
- 下载完关闭下载页前：`1-2` 秒

原则：

- 准确优先
- 稳定优先
- 不为了提速牺牲通过率

## 6. 会话保持原则

下载模块不应该在每篇结束后把整个链路都关掉。

正确方式：

1. 保留代理 CNKI 搜索页
2. 每篇论文单独打开下载页
3. 下载完成后只关闭下载页
4. 回到搜索页继续下一篇

如果后续继续维护下载链路，优先只维护这一条：
